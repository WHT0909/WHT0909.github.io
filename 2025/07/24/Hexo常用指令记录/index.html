## （2）hexo g 或 hexo generate 该指令的主要作用是生成静态网页文件

在本地编写完新文章或者修改了文章内容、主题配置等之后，执行 hexo g 生成最新的静态网页，然后使用 hexo s（hexo server 的缩写，用于启动本地服务器 ）启动本地服务器，在浏览器中访问本地地址（如 http://localhost:4000 ），就能看到最新的博客效果。

hexo g 通常会和其他指令配合使用，比如： - hexo clean：在执行 hexo g 之前，有时会先运行 hexo clean 指令，它会清空 public 文件夹中的内容。这样做是为了确保每次生成的静态文件都是最新的，避免因为之前生成的文件残留而导致的显示异常。 - hexo d：hexo d 是 hexo deploy 的缩写，用于将生成好的静态文件（位于 public 文件夹 ）部署到指定的线上服务（如 GitHub Pages 等 ）。一般在执行完 hexo g 生成静态文件之后，再执行 hexo d 就能把博客发布到线上。不过，使用 hexo d 之前需要先在配置文件中正确配置部署相关的信息，如远程仓库地址、认证信息等。

（3）hexo clean

hexo clean 指令的作用是清理缓存文件和之前生成的静态文件

在执行 hexo g 之前，最好先运行 hexo clean 清理缓存和旧的静态文件，避免因为缓存或旧文件的干扰，导致网页显示的内容与预期不符

（4）hexo s 或 hexo server

用于启动服务，默认地址为4000端口

（5）hexo d 或 hexo deploy

其核心作用是将本地生成的静态文件自动部署到远程服务器或托管平台，实现博客内容的在线发布。
